WDF 1394 Virtual Device Hybrid Sample Driver

Summary

The driver present in this directory is a Windows Driver Foundation port of the 1394 Virtual Device sample driver and supporting dynamic link library present under src\1394 directory.

This document and the associated source code describe how to interact with the 1394 driver stack.  The WDF 1394 Hybrid Sample consist of two sample WDF Drivers, a UMDF component (umdf1394vdev.dll) and a KMDF component (kmdf1394vdev.sys).  The sample also includes a user mode application that instantiates the loading the WDF Hybrid Driver stack and allows interaction with the 1394 bus driver through the installed WDF Hybrid Driver stack.

The umdf1394vdev.dll and kmdf1394vdev.sys source codes provide examples of a Windows Driver Foundation hybrid driver stack interfacing with the upper edge of the core 1394 stack.  In addition to handling asynchronous and isochronous data transfers, the sample source code demonstrates the correct way to use the Windows Driver Foundation to handle; Plug and Play and power management I/O Request Packets (IRPs), which are the basic I/O Manager structures used to communicate with the 1394 stack. See the Microsoft® Windows® WDK for background information on the IRPs discussed in this sample.

The 1394 Virtual Device Hybrid is meant to be a virtual device driver stack that is loaded by using the IOCTL_IEEE1394_API_REQUEST defined in the 1394 DDI section. 

The WDF1394.exe source code provides examples of a user mode application leveraging the WDF 1394 Hybrid stack to provide access to all of the 1394 interfaces.   WDF1394.exe can be used as a sample application which takes advantage of 1394 or as a testing tool to verify the various 1394 interfaces.

Building the Sample

To build the sample, run the build -ceZ command in the DDK build environment.  Samples must be built individually for each component.  The sample will produce three binaries: two WDF drivers (kmdf1394vdev.sys and umdf1394vdev.dll), and one EXE (WDF1394.exe) as well as the corresponding INF file (wdf1394vdev.inf).

Please note that this sample only works on Windows XP and beyond.

Installing the Sample

To install and use the sample 1394 tool set provided a test device must be used which can be loaded virtually via the “Add Test Device” menu option in WDF1394.exe.

Installing a virtual 1394 device:

1)    Copy the wdf1394vdev.inf, kmdf1394vdev.sys, umdf1394vdev.dll and appropriate WDF co installers to a floppy or to a temporary directory on the target machine.

2)    Run WDF1394.EXE on the target machine. Click on “1394 Devices” menu and select “Add Virtual Device”. The application will send the IOCTL_IEEE1394_API_REQUEST to the 1394 bus driver and request it to enumerate a virtual device.

3)    In response to that, you will see “Found New Hardware Wizard” dialog.

4)    Select “Install from a list or specific location (Advanced)”

5)    Select “Don’t search. I will choose the driver to install”.

6)    Click on “Have Disk” and point to the floppy or the directory that contains the INF file and the drivers.

7)    Select “1394 Virtual Device” in the following dialog and click Next.

8)    Click on “Yes” when you get “Security Alert” about installing an unsigned driver.

9)    Finally the system will process the INF file and copy and load the drivers for the device enumerated by the bus.
