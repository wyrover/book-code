LABRADOR 特性示例：实现没有用户接口的服务器

LABRADOR 特性示例展示如何使用 ATL 实现没有任何用户接口的 EXE 服务器。 服务器允许创建支持两个自定义接口（在 Labrador.idl 中定义）的对象。

安全说明

提供此代码示例是为了阐释一个概念，它并不代表最安全的编码实践，因此不应在应用程序或网站中使用此代码示例。

生成并运行示例

该示例使用三个组件：服务器、封送处理 DLL 和驱动程序。

生成并注册组件

  1. 打开解决方案文件 Labrador.sln。

  2. 在“生成”菜单中，单击“生成解决方案”。

    将生成和注册服务器、封送处理 DLL 和驱动程序。

运行驱动程序

  1. 在解决方案资源管理器中，右击 labdriv 项目并选择“设为启动项目”。

  2. 在“调试”菜单中，单击“开始执行(不调试)”。

    驱动程序将创建一个对象，调入该对象几次，然后将其释放。

特性

此示例使用以下特性：

aggregatable、coclass、default、emitidl、helpstring、in、module、object、out、progid、string、threading、uuid、version、vi_progid

_CrtDumpMemoryLeaks; _tcsicmp; _tcstok; _tprintf; _vstprintf; ATLASSERT; BEGIN_COM_MAP; BEGIN_OBJECT_MAP; CComObjectRoot; CoCreateInstance; COM_INTERFACE_ENTRY; CoUninitialize; DECLARE_NOT_AGGREGATABLE; DECLARE_REGISTRY; DispatchMessage; END_OBJECT_MAP; GetCurrentThreadId; GetMessage; OBJECT_ENTRY; OutputDebugString; PostThreadMessage; Trace; va_end; va_list; va_start; wcscpy
