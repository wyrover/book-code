LABRADOR 示例：实现带有进程外自定义接口和编译器 COM 支持的服务器

此示例展示如何使用 ATL 来实现 EXE 服务器。 服务器允许创建支持两个自定义接口（在 Labrador.idl 中定义）的对象。 编译器 COM 支持用于实现执行 C 样式数组的不常用封送处理的自定义接口。

将此示例与 ATL 目录中的 LABRADOR 示例进行比较。

安全说明

提供此代码示例是为了阐释一个概念，它并不代表最安全的编码实践，因此不应在应用程序或网站中使用此代码示例。

生成并运行示例

注释

如果您运行的是 Visual C++ 学习版，则可能需要在运行此示例之前安装 Platform SDK。 有关如何执行此操作的信息，请参见如何：将 Visual C++ 学习版与 Microsoft Platform SDK 一起使用。

生成并运行此示例

  1. 打开解决方案 labrador.sln。

  2. 在“生成”菜单上，单击“生成解决方案”。

  3. 将 labdriv 设置为启动项目（右击该项目节点，然后单击“设为启动项目”）。

  4. 在“调试”菜单上单击“开始执行(不调试)”。
